Referencia técnica de todos los parámetros, variables y fórmulas utilizadas por el sistema de riego inteligente.

Datos Climáticos
ET₀ — Evapotranspiración de referencia wx_et0 Clima mm / día
Cantidad de agua que evaporaría una superficie de pasto de referencia bajo las condiciones climáticas del día. Representa la demanda hídrica atmosférica: a mayor ET₀, mayor calor/viento y más agua consume el cultivo. El sistema la obtiene directamente de Open-Meteo usando el método estándar Penman-Monteith FAO 56, para las coordenadas de Villa Alemana (33.04°S / 71.37°O).
Fuente (API Open-Meteo): GET api.open-meteo.com/v1/forecast
  ?daily=et0_fao_evapotranspiration
  &latitude=-33.04&longitude=-71.37
  &forecast_days=2

// ET₀ del día actual → wx_et0 [mm/d]
Lluvia próximas 24h wx_rain24h Clima mm
Precipitación acumulada prevista para las próximas 24 horas. Se calcula como el máximo entre la lluvia de hoy y la de mañana según el pronóstico de Open-Meteo, tomando el peor caso para decidir si conviene regar.
Derivación en firmware: wx_rain24h = max(rain_today, rain_tomorrow)
// rain_today y rain_tomorrow vienen del campo
// daily=precipitation_sum de Open-Meteo
Temperatura máx./mín. wx_tMax / wx_tMin Clima °C
Temperatura máxima y mínima prevista para hoy. Se obtienen de Open-Meteo (temperature_2m_max / temperature_2m_min). Se muestran en el panel de clima del dashboard como referencia visual.
Parámetros de Suelo
Humedad del suelo moisturePct Suelo %
Porcentaje de humedad medido por el sensor capacitivo del satélite ESP32 (GPIO 34). El satélite envía lecturas vía ESP-NOW cada 30 minutos. El valor se normaliza entre 0% (seco) y 100% (saturado) basándose en valores de referencia del sensor.
Normalización en firmware satélite: raw = analogRead(GPIO34) // 0–4095 (12-bit ADC)
pct = map(raw, WET_VAL, DRY_VAL, 100, 0)
pct = constrain(pct, 0, 100)
Umbral de humedad threshold Suelo %
Porcentaje de humedad por debajo del cual el sistema considera que el suelo está suficientemente seco como para regar. Si moisturePct < threshold, la lógica predictiva evalúa activar el riego. Configurable por zona en la pantalla de Programación. La barra amarilla en el dashboard indica este umbral.
Capacidad hídrica del suelo soilMaxMm Suelo mm
Cantidad máxima de agua que puede retener el suelo en la zona de raíces. Depende del tipo de suelo y profundidad. Valores típicos: arena 60–80 mm, franco 120–150 mm, arcilla 180–220 mm. Se usa para calcular cuánta agua falta para llevar el suelo de la humedad actual hasta el umbral.
Rol en la fórmula de déficit: deficit_mm = soilMaxMm × (threshold − moisturePct) / 100
// ejemplo: 120mm × (80% − 45%) / 100 = 42 mm de déficit
Parámetros de Cultivo
Kc — Coeficiente de cultivo Kc Cultivo adimensional
Factor que ajusta ET₀ al cultivo específico de cada zona. Un Kc de 1.0 significa que el cultivo tiene la misma demanda que la referencia; mayor a 1.0 (ej. tomates en floración: 1.15) consume más; menor (ej. árboles frutales en reposo: 0.5) consume menos.
Valores de referencia FAO 56: Pasto corto / Pradera → Kc ≈ 0.85
Hortalizas (media) → Kc ≈ 1.00
Tomate (floración) → Kc ≈ 1.15
Árbol frutal (reposo) → Kc ≈ 0.50

// Uso en firmware: mm_needed += ET₀ × Kc × 0.5
Parámetros del Sistema de Riego
Caudal de la bomba flowRate Sistema L / h
Volumen de agua que entrega la bomba por hora. Depende del tipo de riego instalado: goteo, microaspersión o aspersión. Se configura en la sección de Programación.
Ejemplos típicos: Goteo (4 goteros/m²) → ~8–16 L/h por zona
Microaspersión → ~40–120 L/h por zona
Aspersión de jardín → ~200–600 L/h por zona
Área de la zona area Sistema m²
Superficie irrigada por la zona en metros cuadrados. Se usa junto con flowRate para calcular la tasa de precipitación del sistema.
Tasa de precipitación del sistema precipR Sistema mm / h
Cuántos milímetros de agua aplica el sistema de riego sobre la zona en una hora. Equivale a la lluvia artificial que genera la bomba. A mayor precipR, menos tiempo de riego se necesita para cubrir el mismo déficit.
Cálculo: precipR [mm/h] = flowRate [L/h] ÷ area [m²]

// Ejemplo: 120 L/h ÷ 15 m² = 8 mm/h
// (1 L sobre 1 m² equivale a 1 mm de lámina)
Eficiencia del sistema efficiency Sistema 0.0 – 1.0
Fracción del agua aplicada que realmente llega a la zona de raíces. El resto se pierde por evaporación, escorrentía o distribución irregular. Un valor de 0.90 significa que el 90% del agua entregada es aprovechada.
Valores típicos: Goteo subsuperficial → 0.90 – 0.95
Goteo superficial → 0.85 – 0.92
Microaspersión cubierta → 0.75 – 0.85
Aspersión al aire libre → 0.65 – 0.75

// Rol en la fórmula: se divide para compensar pérdidas
// mins = (mm_needed / precipR) / efficiency × 60
Lógica de Decisión de Riego
Déficit hídrico necesario mm_needed Decisión mm
Lámina de agua que debe aplicarse para reponer el suelo al nivel del umbral, ajustada por la demanda evapotranspirativa del cultivo y descontando la lluvia prevista. Si mm_needed ≤ 0 tras los ajustes, el sistema no riega.
Fórmula completa (Nivel 2 — Balance Hídrico): // 1. Déficit base del suelo
mm_needed = soilMaxMm × (threshold − hum) / 100

// 2. Ajuste por demanda del cultivo (ET₀ × Kc × 0.5)
mm_needed += wx_et0 × Kc × 0.5

// 3. Cancelar si lluvia prevista ≥ umbral de cancelación
if (wx_rain24h ≥ RAIN_CANCEL_MM) → NO REGAR

// 4. Descontar lluvia parcial (80% de eficiencia de lluvia)
else if (wx_rain24h > 0) → mm_needed −= wx_rain24h × 0.8

// 5. Si tras ajustes mm_needed ≤ 0 → NO REGAR
Duración de riego calculada mins Decisión minutos
Tiempo de riego resultante que aplicará el sistema para cubrir el déficit hídrico calculado, considerando la tasa de precipitación del equipo y su eficiencia.
Fórmula: horas = mm_needed ÷ (precipR × efficiency)
mins = horas × 60

// Ejemplo:
// mm_needed = 18 mm, precipR = 8 mm/h, eff = 0.85
// horas = 18 / (8 × 0.85) = 18 / 6.8 = 2.65 h
// mins = 2.65 × 60 = 159 min

// El resultado se acota entre MIN_MINS (1) y maxMins
Umbral de cancelación por lluvia RAIN_CANCEL_MM Decisión mm
Si la lluvia prevista (wx_rain24h) supera este valor, el sistema cancela el riego programado para esa zona, considerando que la lluvia aportará suficiente agua. Valor configurado en firmware: 5.0 mm.
Lógica en firmware: if (wx_rain24h >= 5.0) {
  result = "Lluvia prevista, sin riego"
  return false // no activa bomba
}
Ventana de ejecución MAX_DECISION_MIN Decisión minutos
Si la hora programada ya pasó, el sistema tiene hasta MAX_DECISION_MIN minutos para ejecutar el riego de todas formas (ej. si el ESP32 estuvo sin WiFi o reiniciándose). Pasado ese tiempo, el intento se cancela. Valor configurado: 30 minutos.
Lógica en predictiveCheck(): minsFrom = (now - sched_time) en minutos
if (minsFrom >= 0 && minsFrom <= 30) {
  → evaluar y regar si corresponde
}
Flujo Completo de Decisión (cada zona)
1
¿Hay hora programada? → No: saltar zona → Sí: continuar
2
¿Estamos en la ventana de ejecución (0–30 min después)? → No: saltar → Sí: continuar
3
¿Ya se regó hoy en este slot? → Sí: saltar (no duplicar) → No: continuar
4
¿Hay bomba activa actualmente? → Sí: esperar (solo 1 bomba a la vez) → No: continuar
5
¿Lluvia prevista ≥ 5 mm? → Sí: cancelar riego → No: continuar
6
¿Humedad actual < Umbral? → No: suelo suficientemente húmedo, no regar → Sí: calcular mm_needed y duración
7
¿mm_needed > 0 tras descontar lluvia? → No: no regar → Sí: ACTIVAR BOMBA por 'mins' calculados
